1 About

webMethods Optimize for SAP enables you to extract process data (such as the start and end time of a process instance), error data, or business data (such as customers, order quantities, and revenues) from a running SAP process. The extracted data can then be sent, using the webMethods SAP Adapter (SAP Adapter) and Integration Server technologies, to webMethods Optimize for Process (Optimize) for business monitoring and analysis.

The new architecture of Optimize for SAP is based on the following principles:

  • Taking the webMethods centric view to SAP business processes... By representing events from a SAP business process as adapter notifications at the webMethods platform, the SAP events can be represented as receive steps that will be managed by the Process Engine. The following mapping between usage patterns and the use of adapter notifications applies:
    • Remote Receiver FM :: Remote event-receiver coupling (asynchronous)
    • Local Receiver FM :: Local event-receiver coupling (asynchronous)

      • Note. The publishable document created with this type of notification will conform to the import parameters defined with RFC WM_PUSH_EVENT. This is because the Local Receiver FM specified under Receiver FM is expected to call RFC WM_PUSH_EVENT in the end.

    • Remote Events :: ALE listener notification (asynchronous)
    • Extending the Receiver FM for process steps not triggered by an event :: RFC listener notification for function name WM_PUSH_EVENT

      • Note. In this case, only one RFC listener notification for function name WM_PUSH_EVENT needs to be created. You can assign it to different steps from your process model by specifying a subscription filter.

      Note. Make sure that the RFC listener notification created for function name WM_PUSH_EVENT is the last one in the list of adapter notifications defined with the listener. You can edit the notification order via Adapters > SAP Adapter > Edit Listener > Edit Notification Order.

  • All Optimize for SAP monitoring will be done via the Process Engine... By uploading the business process for execution and letting the SAP Process Monitoring Agent talking to the Process Engine directly, it is now possible to embed SAP business process steps into an overall business process orchestrated by the Process Engine. Additionally, the already available functionality of Receive Steps, like selecting a correlation service and specifying logged fields, will automatically become available in Optimize for SAP scenarios.
  • Support of event parameters... SAP events can have additional parameters defined with them. By supporting these parameters it will now be possible to monitor business data stored in the SAP event parameter.

2.2 Where can I find the documentation?

Documentation is available from the Software AG Documentation Web site, the Empower Web site, and the Installer:

  • Visit the webMethods area of the Software AG Documentation Web site for the latest updates to the documentation.
  • Visit the Products area of the Empower Web site for documentation for all current and previous versions of our products, and documentation in other languages.
  • Product documentation is also available for download from the Software AG Installer. From the Installer, select "Documentation" to download and install the entire webMethods product library, or navigate within the Documentation folder to select the product libraries to install. By default, the Installer installs the documentation files to a single directory called _documentation, located under your webMethods product installation directory.

3 Samples

3.1 Order to Cash

The sample Order to Cash is the reference implementation chosen to show the benefits of Optimize for SAP. The sample provides a business process model, a package with predefined adapter notifications, and a simulator package.

To install the Order to Cash sample:

  1. Download the sample from the Software AG Developer Community for webMethods.
  2. Install package Order2CashAN from the IntegrationServer/replicate/inbound directory.
  3. Load the process model into Designer and then upload it for execution. This will create a new package Order2CashPE that works together with the notifications defined in package Order2CashAN.

To install the Order to Cash simulator:

  1. Install package Order2CashSC from the IntegrationServer/replicate/inbound directory.

Package Order2CashSC contains service sim:sampleSimulator that can be used to simulate the typical Order to Cash business process without requiring an actual connection to a SAP system resource. Instead it reads and manipulates pipeline files from the files system that were originally created by executing an Order to Cash process instance within SAP.

Most of the SAP related Receive Steps from the Order to Cash process model use the service sample.sap.bem.Correlate:extractCID in conjunction with the Correlation FM z_wm_correlate_documents, installed on your SAP system, to map individual document id's to a business process instance.

With remote event-receiver couplings, the event container will be published to the Broker directly via a matching adapter notification defined on the SAP Adapter.

Local event-receiver couplings are implemented via local receiver FM z_wm_handle_event installed on your SAP system. The event container will be published to the Broker via a matching adapter notification defined on the SAP Adapter by calling RFC WM_PUSH_EVENT at the SAP Adapter from the local receiver FM.

Note. RFC WM_PUSH_EVENT called from the Local Receiver FM does not have to be defined at the calling SAP system. It is already defined in the local SAP Adapter DDIC.

Important! The local event-receiver coupling approach is deprecated. Functionality will not be developed further and may be removed in a later release of Optimize for SAP.
