WebSphere MQ-Level Trace Console Screen

To assist in problem determination with your configured services, WebSphere MQ provides data that the WebSphere MQ Adapter can trace. You use tracing settings when instructed by webMethods Technical Services to locate errors and to troubleshoot problems.

Use the WebSphere MQ-Level Trace Console screen to configure, archive, clear and refresh tracing files.

Specify values for the following fields, or use the following buttons:

Field/Button
Description/Action

WebSphere MQ Trace Status

Select the following trace status for the WebSphere MQ Adapter:

Enabled: Displays tracing messages in the
WebSphere MQ-Level Tracing screen.

Disabled: Does not trace any messages. Default: Disabled

WebSphere MQ Trace Level

Select the level of tracing from 0-5 from the drop-down list, where 0 is the lowest level of tracing.

The adapter automatically outputs all messages at the level you select as well as higher-level messages. For example, if you select 2, the adapter outputs all messages from levels 0 and 1 as well.

Archive Trace File

In the Archive Trace File field, type the name of the file that will contain the trace archive information. Click Archive Trace File and then click OK. The status of the archive trace file displays at the top of the WebSphere MQ-Level Trace Console window as TRACE_ARCHIVED - file=<filename>

Note: The information in the trace file is archived to the fully qualified file name that you specified. If you do not specify a directory path, the file is saved in the webMethods/IntegrationServer directory, where webMethods is the directory in which you installed the webMethods Integration Platform. Also, if the trace file does not already exist, then the WebSphere MQ Adapter creates the file.

Clear Trace File

If you no longer need the tracing file, then you can clear or delete it. To clear tracing files click Clear Trace File and then click OK.

The trace file is cleared from the WebSphere MQ-Level Tracing screen and the status of the file appears at the top of the screen as TRACE_CLEARED. When you clear the trace file from the screen, the contents of the archived trace files are not affected.

Refresh

If you changed any settings on the tracing console or if WebSphere MQ contains any new troubleshooting information, then you need to refresh the tracing file to ensure that the file has the most up-to-date information.

To refresh tracing files, click Refresh. The information in the tracing console is updated with any new settings that you specified or with new information from WebSphere MQ.
