2.0 Known Issues

1-14A8TJ
After installing webMethods WebSphere MQ Adapter 6.5, certain listener and connection parameters configured using webMethods WebSphere MQ Adapter 6.0 do not display their configured values:

- Listener parameter "Convert Application Data" displays as blank.

- Connection parameters "Encoding", "Cache Overridden Connections", and "Action before disconnecting an in-transaction connection" display as blank.

To make these parameter values appear in webMethods WebSphere MQ Adapter 6.5, open each listener or connection in edit mode and save it.

For other known issues and troubleshooting information about webMethods WebSphere MQ Adapter, go to the webMethods Advantage Knowledge Base.

6.0 Globalization

webMethods WebSphere MQ Adapter 6.5 conforms to the internationalization standards of webMethods and includes support for operation in any country, locale or language as specified in the webMethods Installation Guide.

6.1 Certification

webMethods WebSphere MQ Adapter 6.5 was tested with non-ASCII data using various Unicode and non-Unicode encodings. It was tested in common European and Far East Asian locales and is supported for operation in any locale provided by your operating environment. The tested environments included English locale on an English Windows machine and Japanese locale on an English Windows machine.

6.2 Configuration

If you expect to display non-English data in your webMethods tools, make sure an appropriate font is installed in your operating environment. If you see hollow boxes or black squares where you expect to see non-English data, modify your JRE font.properties file to reference an appropriate font. For information on modifying your font.properties file, go to Sun's Java website. For instructions, see:

http://java.sun.com/j2se/1.4.2/docs/guide/intl/fontprop.html

http://java.sun.com/j2se/1.5.0/docs/guide/intl/fontconfig.html

http://www.unicode.org/help/display_problems.html

The configuration of the database you use with webMethods WebSphere MQ Adapter 6.5 affects the characters the database can contain. If you select a non-Unicode encoding for the database, characters from outside that encoding will be lost when you store data in the database. webMethods recommends that you use the most appropriate Unicode encoding for your database character set. See the webMethods Installation Guide and your database vendor's documentation for instructions on choosing and setting the character encoding for your database and environment.

6.3 Localization

webMethods WebSphere MQ Adapter 6.5 is provided with an English user interface, messages, and documentation. Language packs containing support for other languages, when available, can be installed at any time using the webMethods Installer. webMethods WebSphere MQ Adapter 6.5 supports simultaneous multi-locale operation.
